# 一、插件系统规范

## 1. 目录规范（固定结构）

```plaintext
src/plugins/[插件名]/
├── index.ts          # 插件入口（必须）
├── router.ts         # 插件路由（必须）
├── pages/            # 页面
├── components/       # 内部组件
└── stores/           # 状态管理
```

## 2. 命名规范（强制）

- 插件名：**小写英文**，如 `message`、`contact`、`call`
- 路由前缀：`/${插件名}`
- 事件名：`插件名:动作`，如 `message:send`
- store 名：`useXxxStore`，如 `useContactStore`

## 3. 插件接口规范（固定格式）

每个插件必须导出一个 `AppPlugin` 对象：

```typescript
export const 插件名Plugin: AppPlugin = {
  meta: {
    name: '插件名', // 必须唯一
    title: '显示名称',
    icon: 'xxx',
    order: 数字,
  },
  router: 路由对象,
  hooks: {
    install() {}, // 必须
    activate() {}, // 可选
    deactivate() {}, // 可选
    uninstall() {}, // 可选
  },
  dependencies: [], // 依赖插件
};
```

## 4. 路由规范

- 每个插件路由**自动加前缀** `/插件名`
- 页面必须**懒加载**
- 不允许和其他插件路由冲突

## 5. 通信规范

- 插件之间**不能 import 对方**
- 只能通过 `eventBus` 通信
- 事件名必须加插件名前缀：`message:new` `contact:select`

# 二、新增插件的标准步骤（复制即可）

我以 **contact 联系人插件** 为例

## 第 1 步：创建插件目录

```plaintext
src/plugins/contact/
├── index.ts
├── router.ts
├── pages/ContactList.vue
└── stores/contact.ts
```

---

## 第 2 步：写 store（stores/contact.ts）

```typescript
import { defineStore } from 'pinia';
export const useContactStore = defineStore('contact', {
  state: () => ({
    list: [],
  }),
  actions: {
    async init() {},
  },
});
```

---

## 第 3 步：写路由（router.ts）

```typescript
export const contactRouter = {
  prefix: '/contact',
  routes: [
    {
      path: '/',
      name: 'contact-list',
      component: () => import('./pages/ContactList.vue'),
    },
  ],
};
```

---

## 第 4 步：写插件入口（index.ts）【最重要】

```typescript
import type { AppPlugin } from '@/core/plugin/type';
import { pluginManager } from '@/core/plugin';
import { routerManager } from '@/core/router';
import { contactRouter } from './router';

export const contactPlugin: AppPlugin = {
  meta: {
    name: 'contact',
    title: '联系人',
    icon: 'i-heroicons-user-group',
    order: 2,
  },
  router: contactRouter,

  hooks: {
    install(app) {
      routerManager.registerPluginRouter('contact', contactRouter);
      console.log('contact 插件安装');
    },
    activate() {
      console.log('contact 激活');
    },
    deactivate() {
      console.log('contact 失活');
    },
  },
};

pluginManager.register(contactPlugin);
```

---

## 第 5 步：写页面（pages/ContactList.vue）

```Vue
<template>
  <div class="p-4">
    <h1 class="text-xl font-bold">联系人</h1>
  </div>
</template>
```

---

## 第 6 步：在 main.ts 引入插件

```typescript
import './plugins/contact';
```

## ✅ 完成

自动出现：

- 路由 `/contact`
- TabBar 自动多出「联系人」按钮
- 可点击切换
- 刷新不 404
- 插件之间完全隔离

# 三、最精简的新增插件模板

```typescript
// 1. router.ts
export const xxxRouter = {
  prefix: '/xxx',
  routes: [{ path: '/', component: () => import('./pages/XxxPage.vue') }],
};

// 2. index.ts
import type { AppPlugin } from '@/core/plugin/type';
import { pluginManager } from '@/core/plugin';
import { routerManager } from '@/core/router';
import { xxxRouter } from './router';

export const xxxPlugin: AppPlugin = {
  meta: {
    name: 'xxx',
    title: 'XXX',
    icon: 'i-heroicons-xxx',
    order: 3,
  },
  router: xxxRouter,
  hooks: {
    install(app) {
      routerManager.registerPluginRouter('xxx', xxxRouter);
    },
  },
};
pluginManager.register(xxxPlugin);
```

# 四、你只要记住 4 条规则

1.  **插件目录固定**
2.  **插件名唯一**
3.  **路由自动加前缀，不用手改**
4.  **互相通信只用 eventBus**
